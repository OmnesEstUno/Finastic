### Architecture summary

Today's KV keys (all single-user):
```
auth:initialized
auth:passwordHash
auth:totpSecret
data:transactions
data:income
data:userCategories
preauth:<uuid>
```

After this task (usernames as partition key):
```
users:<username>:profile   — { passwordHash, totpSecret, createdAt }
users:<username>:data:transactions
users:<username>:data:income
users:<username>:data:userCategories
meta:initialized           — global flag
meta:usernames             — Set<string> of all usernames (for sign-in validation)
preauth:<uuid>             — unchanged (short-lived token bound to username in payload)
```

The JWT payload adds a `username` field. Every data endpoint reads it from the verified JWT and scopes the KV key accordingly. The auth endpoints accept `username` as input.

**Migration path:** On first boot after upgrade, GET /api/setup/status detects legacy data (`auth:initialized === "true"` and NO `meta:initialized`) and returns `{ initialized: true, migrationPending: true }`. The Login page, on receiving that flag, displays a one-time "Claim your data" form prompting for a username. POST /api/setup/migrate moves the old keys to the new namespaced format and sets `meta:initialized`.

async function authenticate(request: Request, env: Env): Promise<{ username: string } | null> {
  const auth = request.headers.get('Authorization') ?? '';

    if (payload.authenticated !== true || typeof payload.username !== 'string') return null;
    return { username: payload.username };

- [ ] **Step 5: Create `worker/src/migrations.ts`**

```ts
// Migrates single-user KV data into the username-scoped layout.
// Safe to call multiple times — idempotent.
import type { KVNamespace } from '@cloudflare/workers-types';

export async function migrateSingleUserToMultiTenant(
  kv: KVNamespace,
  username: string,
): Promise<{ moved: number }> {
  const legacyKeys = ['auth:passwordHash', 'auth:totpSecret'] as const;
  const legacyDataKeys = ['data:transactions', 'data:income', 'data:userCategories'] as const;
  let moved = 0;

  const [hashVal, totpVal] = await Promise.all(legacyKeys.map((k) => kv.get(k)));
  if (!hashVal || !totpVal) throw new Error('Legacy auth data missing; cannot migrate.');

  const profile = {
    passwordHash: hashVal,
    totpSecret: totpVal,
    createdAt: new Date().toISOString(),
    confirmed: true,
  };
  await kv.put(`users:${username}:profile`, JSON.stringify(profile));
  moved++;

  for (const leaf of legacyDataKeys) {
    const val = await kv.get(leaf);
    if (val) {
      await kv.put(`users:${username}:${leaf}`, val);
      moved++;
    }
  }

  // Update meta
  const existingUsernames = JSON.parse((await kv.get('meta:usernames')) ?? '[]') as string[];
  if (!existingUsernames.includes(username)) {
    existingUsernames.push(username);
    await kv.put('meta:usernames', JSON.stringify(existingUsernames));
  }
  await kv.put('meta:initialized', 'true');

  // Clean up legacy keys last (so migration is restartable if earlier steps fail)
  await Promise.all([
    kv.delete('auth:initialized'),
    kv.delete('auth:passwordHash'),
    kv.delete('auth:totpSecret'),
    ...legacyDataKeys.map((k) => kv.delete(k)),
  ]);

  return { moved };
}
```

### Architecture summary

New KV keys (on top of Task 2):
```
instances:<instance-uuid>                             — { id, name, owner: username, members: [username, ...] }
instances:<instance-uuid>:data:transactions
instances:<instance-uuid>:data:income
instances:<instance-uuid>:data:userCategories
users:<username>:profile                              — now also includes: instanceIds: [uuid, ...], activeInstanceId: uuid
```

**Every endpoint that currently reads `users:<u>:data:*` switches to read `instances:<uuid>:data:*`, where `<uuid>` is derived from the request — a new `X-Instance-Id` header the frontend sends, validated server-side as one the authenticated username is a member of.**

**Migration (continues from Task 2):** When the migration endpoint runs, OR on first login after Task 2 is deployed for already-migrated users, create a default instance named "My Finances" for that user, move their `users:<u>:data:*` keys under `instances:<uuid>:data:*`, and set `activeInstanceId` on the profile.

**Tab UI:** Sticky on the left side of the app (below the navbar). Each tab is a parallelogram — use CSS `clip-path` for the shape. On hover, the hovered tab slides right and expands; others dim slightly. Tabs scroll vertically when overflowing. Adding/renaming/deleting instances happens in Settings.

- [ ] **Step 3: Worker — repartition every data endpoint by instance**

For EVERY data endpoint, introduce an "instance resolver" that:
1. Reads the `X-Instance-Id` header from the request.
2. Validates the authenticated user is a member of that instance.
3. Uses `instances:${instanceId}:data:${leaf}` keys instead of `users:${username}:data:${leaf}`.

```ts
async function resolveInstance(request: Request, auth: { username: string }, env: Env): Promise<{ instanceId: string } | Response> {
  const instanceId = request.headers.get('X-Instance-Id');
  if (!instanceId) return respond({ error: 'Missing instance.' }, 400, corsHeadersDefault);
  const raw = await env.FINANCE_KV.get(`instances:${instanceId}`);
  if (!raw) return respond({ error: 'Instance not found.' }, 404, corsHeadersDefault);
  const inst = JSON.parse(raw) as Instance;
  if (!inst.members.includes(auth.username)) return respond({ error: 'Forbidden.' }, 403, corsHeadersDefault);
  return { instanceId };
}
```

import { useCallback, useEffect, useState } from 'react';
import { Instance } from '../types';
import { getInstances, setActiveInstance, setActiveInstanceIdLocal } from '../api/client';

export function useWorkspaces() {
  const [instances, setInstances] = useState<Instance[]>([]);
  const [activeInstanceId, setActiveInstanceIdState] = useState<string | null>(null);
  const [loading, setLoading] = useState(true);

  const refresh = useCallback(async () => {
    const { instances, activeInstanceId } = await getInstances();
    setInstances(instances);
    setActiveInstanceIdState(activeInstanceId);
    if (activeInstanceId) setActiveInstanceIdLocal(activeInstanceId);
  }, []);

  useEffect(() => { refresh().finally(() => setLoading(false)); }, [refresh]);

  const switchTo = useCallback(async (id: string) => {
    setActiveInstanceIdLocal(id);
    setActiveInstanceIdState(id);
    await setActiveInstance(id);
  }, []);

  return { instances, activeInstanceId, loading, refresh, switchTo };
}
```

import { useEffect } from 'react';

interface ModalProps {
  open: boolean;
  onClose: () => void;
  preventClose?: boolean; // when true, clicking outside does nothing (use during pending edits)
  children: React.ReactNode;
}

export default function Modal({ open, onClose, preventClose, children }: ModalProps) {
  useEffect(() => {
    if (!open) return;
    const onKey = (e: KeyboardEvent) => { if (e.key === 'Escape' && !preventClose) onClose(); };
    window.addEventListener('keydown', onKey);
    document.body.style.overflow = 'hidden';
    return () => { window.removeEventListener('keydown', onKey); document.body.style.overflow = ''; };
  }, [open, onClose, preventClose]);

  if (!open) return null;
  return (
    <div
      className="modal-backdrop"
      onClick={() => { if (!preventClose) onClose(); }}
    >
      <div className="modal-card" onClick={(e) => e.stopPropagation()}>
        <button className="modal-close" onClick={onClose} aria-label="Close">
          <svg width="16" height="16" viewBox="0 0 24 24" fill="none" stroke="currentColor" strokeWidth="2">
            <line x1="18" y1="6" x2="6" y2="18" /><line x1="6" y1="6" x2="18" y2="18" />
          </svg>
        </button>
        {children}
      </div>
    </div>
  );
}

export function getTrackedDuration(transactions: Transaction[]): { years: number; months: number } {
  if (transactions.length === 0) return { years: 0, months: 0 };
  const dates = transactions.map((t) => parseISO(t.date));
  const minDate = new Date(Math.min(...dates.map((d) => d.getTime())));
  const maxDate = new Date(Math.max(...dates.map((d) => d.getTime())));
  const totalMonths = (maxDate.getFullYear() - minDate.getFullYear()) * 12
                    + maxDate.getMonth() - minDate.getMonth();
  return { years: Math.floor(totalMonths / 12), months: totalMonths % 12 };
}
```

```tsx
// frontend/src/components/DateRangePicker.tsx
import { useState } from 'react';
import { parseISO } from 'date-fns';

interface Props {
  value: { start: string; end: string } | null;
  onChange: (range: { start: string; end: string }) => void;
  maxSpanYears?: number;
}
export default function DateRangePicker({ value, onChange, maxSpanYears = 10 }: Props) {
  const [start, setStart] = useState(value?.start ?? '');
  const [end, setEnd] = useState(value?.end ?? '');
  const [error, setError] = useState('');

  function apply() {
    if (!start || !end) { setError('Both dates required.'); return; }
    const startDate = parseISO(start), endDate = parseISO(end);
    if (startDate > endDate) { setError('Start must be before end.'); return; }
    const spanMs = endDate.getTime() - startDate.getTime();
    const spanYears = spanMs / (365.25 * 24 * 60 * 60 * 1000);
    if (spanYears > maxSpanYears) { setError(`Range exceeds ${maxSpanYears}-year limit.`); return; }
    setError('');
    onChange({ start, end });
  }

  return (
    <div style={{ display: 'inline-flex', alignItems: 'center', gap: 8 }}>
      <input type="date" className="input" style={{ padding: '4px 8px', width: 150 }} value={start} onChange={(e) => setStart(e.target.value)} />
      <span style={{ color: 'var(--text-muted)' }}>→</span>
      <input type="date" className="input" style={{ padding: '4px 8px', width: 150 }} value={end} onChange={(e) => setEnd(e.target.value)} />
      <button className="btn btn-sm btn-secondary" onClick={apply}>Apply</button>
      {error && <span className="text-xs text-danger">{error}</span>}
    </div>
  );
}
```
